## Background

- This is the v1.1.0 Stellar GMP release.

Contract changes in the release:

1. Add contractstorage attribute macro, enabling named keys, typed values, storage method specification in the storage layout, storage schema validation, and built-in TTL extension. Use this macro in lieu of Soroban's contracttype macro on storage enums and calling storage on the env.
2. Add only_owner and only_operator macros, reducing boilerplate for auth-checks and increasing code readability. Use this macro in lieu of directly checking the owner or operator.
3. Migrate the axelar-operators contract's storage schema key from 'Operators' to 'Operator'.
4. Add 'stellar/operators.js is-operator', 'stellar/operators.js add-operator', 'stellar/operators.js remove-operator', 'stellar/operators.js collect-fees', 'stellar/operators.js refund', and 'stellar/operators.js execute' commands to interact with the axelar-operators contract.

Deployment changes in the release:

1. Add 'stellar/gateway.js execute' command to execute GMP messages from the CLI.
2. Make gas-token-address and gas-fee-amount optional for 'stellar/gmp.js send' command.
3. Add 'stellar/contract.js paused', 'stellar/contract.js pause', 'stellar/contract.js unpause' commands to check the pausable state of a Stellar contract.
4. Make minimum-rotation-delay optional for 'stellar/deploy-contract.js' commands.
5. Add 'evm/gateway.js --action isContractCallApproved' command to verify if a gateway contract call is approved from the CLI.
6. Add 'evm/gateway.js --action isMessageApproved' command to verify if a gateway message is approved from the CLI.

## Deployment

Create an `.env` config. `CHAIN` should be set to `stellar` for mainnet, and `stellar-2025-q1` for all other networks.

```yaml
PRIVATE_KEY=xyz
ENV=xyz
CHAIN=xyz
```

## Checklist

The following checks should be performed after the rollout:

### Axelar Operators Calls

1. Verify the axelar_operators storage has migrated as expected.

| Network              | `OPERATOR_ADDRESS`|
| -------------------- | ---------------------------------------------------------- |
| **Devnet-amplifier** | `GBM47ULSN4OEQNPECDIMZSDMK6AH2RNICV2VGOYPBVJ4LBFCFFQP2BXM` |
| **Stagenet**         | `TBD`                                                      |
| **Testnet**          | `TBD`                                                      |
| **Mainnet**          | `TBD`                                                      |

Run the following command for each OPERATOR_ADDRESS:
```bash
node stellar/operators.js is-operator OPERATOR_ADDRESS_A
node stellar/operators.js is-operator OPERATOR_ADDRESS_B
node stellar/operators.js is-operator OPERATOR_ADDRESS_C
```

Every call should return true.

### Stellar → EVM GMP Call

1. Send a GMP call

```bash
node stellar/gmp.js send [destination-chain] [destination-address] 0x1234
```

2. Route GMP call via Amplifier

- https://docs.axelar.dev/dev/amplifier/chain-integration/relay-messages

3. Submit proof with multisig session id

```bash
# Change `PRIVATE_KEY and `ENV` in `.env` from Stellar to EVM
node evm/gateway.js -n [destination-chain] --action submitProof --multisigSessionId [multisig session id]
```

4. Confirm whether the message is approved

```bash
node evm/gateway.js -n [destination-chain] --action isContractCallApproved --commandID [command-id] --sourceChain $CHAIN --sourceAddress [source-address] --destination [destination-address] --payloadHash 0x1ac7d1b81b7ba1025b36ccb86723da6ee5a87259f1c2fd5abe69d3200b512ec8
```

### EVM -> Stellar GMP Call

1. Send a GMP call

```bash
node evm/gateway.js -n [destination-chain] --action callContract --destinationChain stellar --destination [destination-address] --payload 0x1234
```

2. Route GMP call via Amplifier

- https://docs.axelar.dev/dev/amplifier/chain-integration/relay-messages

3. Submit proof with multisig session id

```bash
# Change `PRIVATE_KEY and `ENV` in `.env` from EVM to Stellar
node stellar/gateway.js submit-proof [multisig session id]
```

4. Call execute to validate message

```bash
node stellar/gmp.js execute [destination-chain] [message-id] [source-address] [0x1234]
```
