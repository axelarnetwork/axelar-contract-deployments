## Background

This document covers releases v1.14.0 through v1.14.2. Key improvements across these releases include:

### v1.14.0 (2025-12-10)

This release introduced modular handler broadcasting support and event verifier functionality:

- **Modular Handler Broadcasting**: Enables broadcasting for modular handlers, improving the architecture for handler-based operations
- **Event Verifier**: Adds comprehensive event verifier support with configurable attributes and event filtering
- **Handler Migrations**: Includes stellar and sui handler migrations
- **SDK 0.50 Support**: Backport support for SDK 0.50 upgrade
- **Multisig Handler**: Adds multisig module handler support
- **EVM Handler Improvements**: Unifies modular EVM handlers and adds message verifier handler
- **Bug Fixes**: Prevents gRPC server from hanging during shutdown

### v1.14.1 (2025-12-15)

This release added XRPL support and backported changes:

- **XRPL Voting Handler**: Added XRPL voting handler support
- **Backport Changes**: Backported releases/v1.13.x changes to main

### v1.14.2 (2025-12-19)

This release focuses on code cleanup and handler migrations:

- **Code Cleanup**: Removed event_processor and legacy handlers to streamline the codebase
- **Solana Handler Migration**: Migrated Solana ampd handler to the new architecture

## Deployment

Restart `ampd` with the new binary. Binaries can be found at [axelar-amplifier/releases/tag/ampd-v1.14.2](https://github.com/axelarnetwork/axelar-amplifier/releases/tag/ampd-v1.14.2).

### Checklist

Run the below command to output the version:

```sh
$ ampd --version
ampd 1.14.2
```

Check `ampd` logs to ensure it restarts fine. Monitor voting and signing for your verifier on axelarscan to verify it's operating correctly.
